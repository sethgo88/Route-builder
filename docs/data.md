# Route Builder — Data Reference

> **Keep this file current when:** adding a column to `routes` or `settings` in `db.ts` · adding/removing a function from `db.ts` · changing the Supabase table schema · changing the sync strategy (push/pull logic in `syncService.ts`) · adding a new type to `routeStore.ts` or `db.ts`.

## Core TypeScript Types

Defined in `src/store/routeStore.ts` and `src/services/db.ts`.

```ts
// routeStore.ts
interface Coordinate { longitude: number; latitude: number; }

interface Waypoint {
  id: string;
  coordinate: Coordinate;
  snapAfter: boolean;  // segment routing mode into this waypoint
}

interface RouteStats { distanceKm: number; gainM: number; lossM: number; }

type EditingMode = 'view' | 'creating' | 'editing';

// db.ts
interface SavedRoute {
  id: number;               // SQLite INTEGER PK
  name: string;
  color: string;            // hex, default '#3b82f6'
  waypoints: Waypoint[];    // deserialized from JSON column
  geometry: Feature<LineString>;  // deserialized from JSON column
  stats: RouteStats | null;
  createdAt: string;        // ISO timestamp
}

interface RouteForSync {
  localId: number;
  remoteId: string | null;  // Supabase UUID, null until first sync
  name: string;
  waypoints: Waypoint[];
  geometry: Feature<LineString>;
  stats: RouteStats | null;
  createdAt: string;
  deletedAt: string | null;
}
```

## SQLite Schema

Database file: `routes.db` (opened via `expo-sqlite`).

### `routes` table

| Column | Type | Notes |
|---|---|---|
| `id` | `INTEGER PRIMARY KEY AUTOINCREMENT` | Local row ID |
| `name` | `TEXT NOT NULL` | User-given name |
| `waypoints` | `TEXT NOT NULL` | `JSON.stringify(Waypoint[])` |
| `geometry` | `TEXT NOT NULL` | `JSON.stringify(Feature<LineString>)` |
| `stats` | `TEXT` | `JSON.stringify(RouteStats)` or NULL |
| `created_at` | `TEXT NOT NULL` | ISO timestamp |
| `remote_id` | `TEXT` | Supabase UUID; NULL until first successful push |
| `deleted_at` | `TEXT` | ISO timestamp for soft-delete; NULL = active |
| `updated_at` | `TEXT` | ISO timestamp, updated on every write |
| `color` | `TEXT NOT NULL DEFAULT '#3b82f6'` | Hex line colour |

Columns after `created_at` were added via idempotent `ALTER TABLE` migrations in `initDb()`. Always filter `WHERE deleted_at IS NULL` on reads.

### `settings` table

| Column | Type | Notes |
|---|---|---|
| `key` | `TEXT PRIMARY KEY` | Setting name (e.g. `'unit_system'`) |
| `value` | `TEXT NOT NULL` | Setting value |
| `updated_at` | `TEXT NOT NULL` | ISO timestamp for sync comparison |

Seeded on first run: `unit_system = 'metric'`.

## db.ts Public API

All functions are synchronous (expo-sqlite sync API).

### CRUD

| Function | Signature | Returns |
|---|---|---|
| `initDb()` | `() => void` | Creates tables + runs migrations. Call once on app start. |
| `saveRoute(name, color, waypoints, geometry, stats)` | `(string, string, Waypoint[], Feature<LineString>, RouteStats\|null) => number` | `lastInsertRowId` |
| `updateRoute(id, name, color, waypoints, geometry, stats)` | `(number, string, string, Waypoint[], Feature<LineString>, RouteStats\|null) => void` | |
| `listRoutes()` | `() => SavedRoute[]` | All non-deleted, ORDER BY created_at DESC |
| `getRoute(id)` | `(number) => SavedRoute \| null` | Single route or null |
| `deleteRoute(id)` | `(number) => string \| null` | Soft-delete; returns `remote_id` for cloud propagation |
| `getSetting(key)` | `(string) => string \| null` | |
| `setSetting(key, value)` | `(string, string) => void` | Upsert with current timestamp |
| `getSettingRow(key)` | `(string) => { value: string; updatedAt: string } \| null` | Includes timestamp for sync comparison |

### Sync Helpers (used by syncService only)

| Function | Returns | Purpose |
|---|---|---|
| `getRouteForSync(localId)` | `RouteForSync \| null` | Full row including sync fields |
| `markRouteSynced(localId, remoteId)` | `void` | Stores Supabase UUID after successful push |
| `countUnsyncedRoutes()` | `number` | Routes with `remote_id IS NULL AND deleted_at IS NULL` |
| `listUnsyncedRouteIds()` | `number[]` | IDs of unsynced routes |
| `listLocalRemoteIds()` | `string[]` | All known `remote_id` values (for pull dedup) |
| `insertRemoteRoute(route)` | `void` | Insert a route downloaded from Supabase; no-op if `remote_id` already exists |

## Sync Strategy

**Local-first.** All writes go to SQLite first. Supabase calls are fire-and-forget; failures are `console.warn` in `__DEV__` and never surfaced to the user.

**Push flow:**
- `pushRoute(localId)` reads the route via `getRouteForSync`, checks auth, upserts on Supabase `routes` table (conflict target: `id`). On success, stores returned UUID via `markRouteSynced`.
- If not signed in, `pushRoute` exits silently.

**Pull flow:**
- `pullMissingRoutes()` — called once after sign-in. Fetches all non-deleted remote routes, skips any whose `remote_id` already exists locally (`listLocalRemoteIds()`), inserts the rest via `insertRemoteRoute()`.

**Settings sync:**
- `pullSettings(applyFn)` — last-writer-wins by `updated_at`. Calls `applyFn(key, value)` for each remote setting that is newer than local.
- `pushSetting(key, value)` — upserts on `(user_id, key)` conflict.
